# Snipaste 截图功能调研与需求文档

## 1. 文档信息

- 调研日期：2026-07-09
- 调研对象：Snipaste 2.x 截图相关能力
- 文档目标：基于 Snipaste 的公开资料，抽象出“高效率桌面截图工具”的产品需求，作为后续功能设计、研发拆解和验收依据。
- 资料范围：优先使用 Snipaste 官网、GitHub Wiki、下载/更新日志、隐私政策等官方资料；未在官方资料中确认的能力不作为核心需求。

## 2. 调研结论摘要

Snipaste 的截图体验不是单点“截一张图”，而是围绕“截图、标注、输出、复用”的连续工作流设计。其核心优势体现在：

1. 快速入口：默认通过全局快捷键 `F1` 开始截图，支持托盘图标入口。
2. 精准选择：支持窗口/界面元素检测、全屏、上次截图区域、固定区域、指定尺寸、延迟截图等选择方式。
3. 像素级控制：截图中可使用键盘微调光标与截图区域，配合放大镜、颜色拾取提高精度。
4. 即时标注：截图阶段即可进行矩形、椭圆、箭头、画笔、马克笔、文字、马赛克、模糊、橡皮等标注。
5. 多输出链路：截图可复制到剪贴板、保存文件、快速保存、贴到屏幕、打印、自动保存，也可通过命令行组合输出。
6. 历史复用：成功截图会进入历史记录，支持前后切换复用，降低重复截取成本。
7. 本地优先：官方隐私政策说明截图和图片处理保留在本地设备，网络访问主要用于用户触发下载、更新检查、专业版授权等场景。

产品实现上，MVP 应优先保证“快捷键启动、区域/窗口/元素识别、像素级微调、基础标注、复制/保存/贴图输出、历史记录”闭环；专业化能力可再扩展 OCR、二维码识别、命令行、固定比例、刷新截图、超级截图、多屏/DPI 深度优化等。

## 3. 产品定位

### 3.1 一句话定位

面向桌面办公、研发、设计、测试和内容创作用户的高效率截图工具，帮助用户在不中断当前工作流的情况下完成精准截图、轻量标注、快速输出和临时视觉参考。

### 3.2 目标用户

- 研发人员：截取 UI、日志、错误状态，快速标注问题点并复制到 IM、Issue 或文档。
- 测试人员：复现缺陷时截取窗口/控件区域，进行箭头、框选、模糊敏感信息后提交。
- 设计/产品人员：截取界面局部，做对比、批注、颜色拾取和临时参考。
- 内容/文档人员：截取操作步骤、菜单、弹窗或配置界面，用于教程和说明文档。
- 日常办公用户：用最短路径截取屏幕信息并发送、保存或固定在屏幕上。

### 3.3 核心价值

- 快：快捷键触发，少弹窗，输出路径短。
- 准：自动识别窗口/控件，支持像素级微调。
- 轻：标注与输出在截图流程内完成，不强依赖外部图片编辑器。
- 连贯：截图后可复制、保存、快速保存、贴图、打印或进入自动化链路。
- 私密：截图内容默认本地处理，降低敏感截图外传风险。

## 4. 功能范围

### 4.1 本期范围

- 截图启动与退出
- 截图区域选择
- 窗口/界面元素检测
- 多屏与高 DPI 基础支持
- 鼠标光标捕获开关
- 放大镜与颜色拾取
- 像素级移动/调整
- 截图历史回放
- 基础图片标注
- 输出到剪贴板、文件、快速保存、贴图窗口
- 快捷键配置
- 本地隐私与权限提示

### 4.2 暂不纳入 MVP

- 滚动截图/长截图：官方 FAQ 将 scrolling capture 归入进展类问题，公开资料未显示其为当前核心截图能力。
- GIF/视频录制：与静态截图工作流不同，单独立项更合理。
- 云端同步、团队素材库、在线协作：偏资产管理，不属于 Snipaste 截图功能的核心闭环。
- 重型图片编辑：复杂图层、滤镜、海报排版等应交由专业编辑器处理。

## 5. 用户故事

### 5.1 快速截取并发送

作为用户，我希望按下快捷键后选择屏幕区域，双击或回车即可复制到剪贴板，方便立即粘贴到聊天工具或文档。

验收标准：

- 按默认截图快捷键进入截图模式。
- 鼠标拖拽生成截图区域。
- 按 `Enter`、`Ctrl+C` 或双击区域后，截图写入系统剪贴板。
- 截图模式自动退出。

### 5.2 精准截取某个控件

作为研发/测试用户，我希望截图工具自动识别窗口和界面元素，避免手工拖拽误差。

验收标准：

- 鼠标移动到窗口或控件上时，高亮可截图区域。
- 支持在窗口检测和界面元素检测之间切换。
- 用户确认后，选区边界与识别区域一致。
- 识别失败时允许手动拖拽区域兜底。

### 5.3 截图时进行像素级调整

作为设计/文档用户，我希望用键盘微调截图框，确保边缘对齐且不多截/漏截。

验收标准：

- 支持键盘移动光标或截图区域。
- 支持按 1 像素扩大或缩小截图区域。
- 调整过程中实时更新选区边框和尺寸信息。
- 调整操作不破坏当前标注内容。

### 5.4 截图后轻量标注

作为测试/产品用户，我希望在截图流程中直接加框、箭头、文字、马赛克或模糊，不需要再打开图片编辑器。

验收标准：

- 截图区域确定后可打开标注工具条。
- 支持基础形状、画笔、文本、马赛克、模糊、橡皮。
- 支持撤销、重做、清空编辑。
- 标注完成后可复制、保存或贴图。

### 5.5 截图历史复用

作为需要重复截取同一区域的用户，我希望快速回到上一张成功截图区域或浏览截图历史。

验收标准：

- 成功截图进入历史记录。
- 进入截图模式后可切换上一条/下一条历史记录。
- 支持“以上次成功截图区域开始截图”。
- 可配置历史记录数量上限。

### 5.6 截图后固定在屏幕上

作为设计/研发用户，我希望把截图作为置顶浮窗固定在屏幕上，作为对照参考。

验收标准：

- 截图确认后支持直接贴到屏幕。
- 贴图窗口默认置顶，可移动。
- 贴图窗口支持缩放、透明度调整、关闭/销毁。
- 贴图窗口不影响源截图的剪贴板/保存输出。

## 7. 交互流程

### 7.1 基础截图流程

1. 用户按截图快捷键或点击托盘入口。
2. 系统进入截图模式，冻结当前屏幕画面并显示半透明遮罩。
3. 用户通过自动识别区域或手动拖拽确定截图范围。
4. 用户可进行像素级调整、颜色拾取或标注。
5. 用户选择输出：复制、保存、快速保存、贴图、打印。
6. 系统执行输出动作，记录成功截图历史。
7. 系统退出截图模式。

### 7.2 取消截图流程

1. 用户进入截图模式。
2. 用户按 `Esc`、点击关闭按钮或在非编辑状态右键。
3. 系统取消截图，不写入成功截图历史。
4. 如开启“记录取消截图过程”，则写入扩展历史。

### 7.3 截图后贴图流程

1. 用户在截图模式中选择“贴到屏幕”。
2. 系统将截图生成独立浮窗。
3. 浮窗默认置顶展示，可拖拽移动。
4. 用户可缩放、调整透明度、标注、隐藏、关闭或销毁浮窗。

## 9. 非功能需求

### 9.1 性能

- 截图快捷键触发到进入截图模式的目标耗时不超过 200ms。
- 常规 4K 单屏截图选区拖拽、放大镜、边框刷新应保持流畅，不出现明显掉帧。
- 多屏截图模式下，屏幕坐标、缩放比例和选区边界应一致，不应出现错位。
- 马赛克、模糊、阴影等绘制操作应具备 GPU 加速或降级策略。

### 9.2 兼容性

- 支持 Windows 10/11，优先兼容多屏、不同 DPI 缩放、混合缩放场景。
- 后续可扩展 macOS、Linux；Linux 需单独考虑 Wayland 截图权限。
- 支持高 DPI/Retina 显示，避免截图模糊、工具条尺寸异常。
- 支持常见输入法场景，尤其是文字标注中的中日韩输入。

### 9.3 隐私与安全

- 截图、标注、贴图、历史记录默认本地处理和保存。
- 不默认上传截图内容到云端。
- 若引入 OCR、翻译、云同步等联网能力，必须明确提示数据流向，并提供关闭选项。
- 历史记录和快速保存目录需允许用户查看、修改和清理。
- 对可能包含敏感信息的截图，提供马赛克/模糊等脱敏工具。

### 9.4 稳定性

- 快捷键注册失败时给出明确失败原因。
- 屏幕插拔、DPI 变化、显卡异常等情况下，截图模块应能恢复或提示重启。
- 截图模式中异常退出不应影响剪贴板已有内容。
- 保存失败时保留截图状态，允许用户选择其他输出方式。

### 9.5 可用性

- 核心路径应不依赖复杂配置，首次启动可直接使用默认快捷键。
- 高级配置默认隐藏，避免普通用户被配置项淹没。
- 所有快捷键均应可在偏好设置中查看和修改。
- 标注工具条应可通过键盘快速显示/隐藏，避免遮挡截图内容。

## 10. 数据与存储需求

### 10.1 本地数据

- 配置数据：快捷键、输出目录、默认格式、历史数量、标注样式、是否捕获光标等。
- 截图历史：成功截图缩略图/原图、时间、区域坐标、屏幕信息、输出状态。
- 快速保存文件：用户指定目录下的图片文件。
- 贴图状态：可选保存贴图窗口位置、尺寸、透明度、分组信息。

### 10.2 数据清理

- 支持清空截图历史。
- 支持设置历史记录上限，超过后自动淘汰最旧记录。
- 支持修改历史目录和快速保存目录。
- 支持在卸载或退出时由用户选择是否清理本地数据。

## 11. 验收指标

| 指标 | 验收口径 |
|---|---|
| 快捷启动成功率 | 常规桌面环境下连续 100 次快捷键启动，无失败、卡死或错位。 |
| 选区精度 | 通过键盘微调后，选区边界与目标像素误差不超过 1px。 |
| 输出成功率 | 复制、保存、快速保存、贴图四条 MVP 输出链路均可稳定完成。 |
| 多屏兼容 | 主屏/副屏、不同缩放比例下截图坐标正确。 |
| 历史复用 | 成功截图可进入历史，并可在截图模式中切换回放。 |
| 隐私默认 | 无用户主动触发时，截图内容不得离开本地设备。 |
| 易用性 | 新用户无需配置即可完成“截图、标注、复制”主流程。 |

## 12. 风险与约束

- 多屏高 DPI 是桌面截图工具的高风险区域，需尽早做兼容性验证。
- 界面元素检测依赖系统 API 和应用框架，不同应用识别质量会有差异，必须提供手动选区兜底。
- 全局快捷键易与系统或其他工具冲突，需要冲突检测和清晰提示。
- 贴图窗口、鼠标穿透、置顶、虚拟桌面等能力在不同系统版本上行为差异较大。
- OCR、二维码识别、命令行自动化属于增强能力，不应阻塞 MVP 截图闭环。

## 13. 里程碑建议

### M1：基础截图闭环

- 快捷键/托盘启动截图
- 手动区域选择、窗口检测、元素检测
- 复制、保存、快速保存
- 取消截图
- 基础设置页

### M2：精准与标注

- 放大镜、颜色拾取、像素级调整
- 基础标注工具
- 撤销/重做
- 多屏与高 DPI 基础适配

### M3：历史与贴图

- 成功截图历史
- 上次区域/历史回放
- 截图贴到屏幕
- 贴图窗口缩放、透明度、关闭/销毁

### M4：自动化与专业能力

- 命令行截图与输出
- 延迟截图、固定尺寸/比例
- OCR、二维码识别
- 刷新截图、超级截图
- 更完整的热键和高级配置
